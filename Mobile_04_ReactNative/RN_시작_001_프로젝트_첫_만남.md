## 📖 본문

### 1. 프로젝트가 뭐야?

#### 한 줄 요약
**"웹사이트를 앱으로 포장한 하이브리드 앱"**

#### 자세한 설명
이 앱은 **업무 관리 시스템**의 모바일 앱입니다.

**핵심 특징:**
- 실제 콘텐츠는 **웹(HTML/CSS/JavaScript)**으로 만들어져 있음
- React Native는 **껍데기(컨테이너)** 역할만 함
- 앱 스토어에 올리려면 네이티브 앱이어야 해서 RN을 사용

**비유로 이해하기:**
```
🌐 웹사이트 (실제 콘텐츠)
   ↓
📦 React Native (포장지)
   ↓
📱 App Store/Play Store (선물 배달)
```

웹사이트를 그냥 브라우저로 보면 되잖아요? 왜 앱으로 만들까요?

**앱의 장점:**
1. **푸시 알림** 받을 수 있음
2. **카메라, 위치 정보** 같은 하드웨어 사용 가능
3. **생체 인증** (지문, Face ID) 사용 가능
4. **앱 아이콘**을 홈 화면에 배치 → 접근성 UP
5. **오프라인에서도** 일부 기능 동작 가능

### 2. 왜 React Native를 선택했을까?

#### 선택지가 3가지 있었습니다

**Option 1: 네이티브 앱 (Swift/Kotlin)**
```
iOS: Swift로 개발
Android: Kotlin으로 개발
```
- ✅ 성능 최고
- ❌ 같은 기능을 2번 만들어야 함 (비용 2배)
- ❌ iOS 개발자 + Android 개발자 둘 다 필요

**Option 2: React Native (선택!)**
```
하나의 코드로 iOS + Android 동시 개발
```
- ✅ 한 번 개발하면 양쪽 다 됨
- ✅ 웹 개발자도 쉽게 배울 수 있음
- ✅ 성능도 준수함
- ❌ 네이티브보다는 조금 느림

**Option 3: Flutter**
```
Google이 만든 크로스플랫폼 프레임워크
```
- ✅ 성능 좋음
- ❌ 새로운 언어(Dart) 배워야 함
- ❌ 커뮤니티가 RN보다 작음

**결론: React Native 선택 이유**
1. JavaScript 사용 → 웹 개발자가 바로 투입 가능
2. 한 번 개발로 iOS + Android 커버
3. 커뮤니티가 크고 라이브러리 많음

### 3. WebView 기반 아키텍처가 뭔데?

#### 일반 React Native 앱 vs 웹뷰기반 앱

**일반 RN 앱 (Instagram, Discord 등):**
```
React Native로 UI 전부 구현
   ↓
<View>, <Text>, <Button> 같은 RN 컴포넌트 사용
   ↓
네이티브 UI로 변환
```

**WebView 기반 앱:**
```
React Native는 껍데기만
   ↓
<WebView> 컴포넌트 하나만 띄움
   ↓
그 안에서 웹사이트 로딩
```

#### 왜 이렇게 만들었나?

**배경 스토리:**
1. 이미 웹사이트가 완성되어 있었음
2. 앱도 만들어야 하는데, 똑같은 기능을 또 만들기는 비효율적
3. 해결책: **웹을 앱 안에 넣자!**

**장점:**
- ✅ 웹 개발팀이 기능 업데이트하면 앱도 자동 반영
- ✅ 앱 심사 없이 즉시 배포 가능 (웹만 바꾸면 됨)
- ✅ 하나의 코드베이스 유지 (웹 + 앱)

**단점:**
- ❌ 네이티브 앱보다 느릴 수 있음
- ❌ 네트워크 없으면 대부분 기능 안 됨
- ❌ 앱스러운 UX 구현이 어려움

#### 아키텍처 다이어그램

```
┌─────────────────────────────────────┐
│         React Native App            │  ← 껍데기
│  ┌───────────────────────────────┐  │
│  │        <WebView>              │  │
│  │  ┌─────────────────────────┐  │  │
│  │  │  웹사이트 (HTML/JS)      │  │  │  ← 실제 콘텐츠
│  │  │  - 로그인 화면           │  │  │
│  │  │  - 상품 목록             │  │  │
│  │  │  - 주문 화면             │  │  │
│  │  └─────────────────────────┘  │  │
│  └───────────────────────────────┘  │
│                                     │
│  Native Modules (카메라, 위치 등)    │  ← 네이티브 기능
└─────────────────────────────────────┘
```

**핵심 포인트:**
- 사용자가 보는 화면 = 웹
- React Native = 웹을 감싸는 컨테이너 + 네이티브 기능 제공

### 4. 내가 수정할 코드는 어디에 있나?

#### 프로젝트 폴더 구조 (간단 버전)

```
myapp_source/
├── 📱 android/           ← Android 네이티브 코드
├── 📱 ios/              ← iOS 네이티브 코드
├── 📦 node_modules/     ← 설치된 패키지들 (건드리지 마세요!)
├── ⚛️ src/              ← 여기가 메인 작업 공간!
│   ├── components/      ← 재사용 가능한 UI 조각들
│   ├── screens/         ← 화면 컴포넌트들
│   ├── webview/         ← WebView 관련 로직 (핵심!)
│   ├── lib/            ← 유틸리티 함수들
│   └── navigations/    ← 화면 전환 로직
├── 🎨 App.tsx           ← 앱 시작점 (진입점)
├── 📝 package.json      ← 프로젝트 설정 파일
└── ⚙️ .env.dev/.env.stg/.env.prd  ← 환경 설정
```

#### 신입 개발자가 가장 많이 수정할 파일들

**1순위: src/webview/ 폴더**
```
src/webview/
├── handlers/
│   ├── onMessage.ts       ← 웹 ↔ 앱 메시지 처리 (가장 중요!)
│   ├── onNavStateChange.ts
│   └── onShouldStart.ts
├── utils/
│   ├── sendToWeb.ts       ← 앱 → 웹 메시지 전송
│   ├── biometric.ts       ← 생체 인증
│   ├── visionCamera.ts    ← 카메라
│   └── geoLocation.ts     ← 위치
└── constants.ts           ← WebView 설정 (시작 URL 등)
```

**2순위: src/screens/ 폴더**
```
src/screens/
├── WebShell.tsx           ← 메인 WebView 화면
├── PermissionGuide.tsx    ← 권한 안내 화면
└── socialLogin/
    └── WebPopup.tsx       ← 소셜 로그인 팝업
```

**3순위: src/lib/ 폴더**
```
src/lib/
├── analytics.ts           ← Google Analytics (GA4)
├── messaging.ts           ← 푸시 알림 (FCM)
├── secureStore.ts         ← 보안 저장소 (토큰 등)
└── socialStore.ts         ← 전역 상태 관리
```

#### 네이티브 코드는 언제 수정하나?

**거의 안 건드림 (시니어가 함):**
- `android/` - 새로운 Native Module 추가할 때만
- `ios/` - 새로운 Native Module 추가할 때만

**가끔 수정:**
- `android/app/build.gradle` - 버전 변경, 라이브러리 추가
- `ios/MyApp/Info.plist` - 권한 추가, 설정 변경

### 5. 코드 한 줄 읽어보기

#### App.tsx 열어보기

```typescript
// App.tsx (앱 진입점)

import React from 'react';
import StackNavigation from './src/navigations/Stack';

const App = () => {
  // 앱이 시작되면 이 컴포넌트가 제일 먼저 실행됨

  return (
    <StackNavigation />  // 화면 전환 관리자를 렌더링
  );
};

export default App;
```

**한 줄 요약:**
- `App.tsx` → `StackNavigation` → `WebShell.tsx` → WebView 순서로 실행됨

## 💡 직접 해보기

### 실습 1: 프로젝트 폴더 탐색하기

1. **VS Code로 프로젝트 열기**
   ```bash
   code myapp_source/
   ```

2. **README.md 읽어보기**
   - 프로젝트 전체 설명이 있음
   - 설치 방법, 실행 방법 확인

3. **package.json 열어보기**
   - `dependencies`: 사용 중인 라이브러리 목록
   - `scripts`: 실행 가능한 명령어들

   ```json
   {
     "scripts": {
       "android:dev": "react-native run-android --mode=devDebug",
       "ios:dev": "react-native run-ios"
     }
   }
   ```

4. **src/screens/WebShell.tsx 열어보기**
   - 몇 줄이나 되는지 확인 (200줄 정도)
   - `<WebView>` 찾아보기 (Cmd+F / Ctrl+F)

### 실습 2: 파일 개수 세어보기

```bash
# src 폴더 안의 TypeScript 파일 개수
find src -name "*.ts" -o -name "*.tsx" | wc -l
```

**예상 결과:** 약 30-35개

## 🐛 자주 하는 실수

### ❌ 실수 1: node_modules 폴더를 Git에 커밋
```
❌ 안 돼요!
node_modules 폴더는 절대 Git에 올리면 안 됨
→ 용량 엄청 큼 (수백 MB)
→ package.json만 있으면 npm install로 복구 가능
```

### ❌ 실수 2: .env 파일을 Git에 커밋
```
❌ 안 돼요!
.env 파일은 환경 변수 (비밀번호, API 키 등)
→ 보안상 위험
→ .gitignore에 추가되어 있음
```

### ❌ 실수 3: iOS와 Android 코드를 헷갈림
```
❌ iOS에서 작업하다가 Android 폴더 수정
→ 플랫폼 확인하고 작업하기!
```
