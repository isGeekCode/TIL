# Design Patterns - 디자인 패턴

> 재사용 가능한 설계 솔루션

**(예정중)**

## 개요
디자인 패턴은 소프트웨어 설계에서 자주 발생하는 문제에 대한 검증된 해결책입니다.

## GoF 23가지 패턴

### 생성 패턴 (Creational Patterns)
- Singleton - 단 하나의 인스턴스
- Factory Method - 객체 생성을 서브클래스에 위임
- Abstract Factory - 관련된 객체군 생성
- Builder - 복잡한 객체 생성
- Prototype - 복제를 통한 객체 생성

### 구조 패턴 (Structural Patterns)
- Adapter - 인터페이스 변환
- Bridge - 추상과 구현 분리
- Composite - 트리 구조
- Decorator - 동적 기능 추가
- Facade - 단순화된 인터페이스
- Flyweight - 공유를 통한 메모리 절약
- Proxy - 대리자

### 행위 패턴 (Behavioral Patterns)
- Observer - 이벤트 알림
- Strategy - 알고리즘 캡슐화
- Command - 요청을 객체화
- State - 상태에 따른 행위 변경
- Template Method - 알고리즘 골격 정의
- Iterator - 순차적 접근
- 등...
