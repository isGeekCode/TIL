# Refactoring - 리팩토링

> 코드 개선 기법

**(예정중)**

## 개요
리팩토링은 코드의 외부 동작은 그대로 유지하면서 내부 구조를 개선하는 작업입니다.

## 언제 리팩토링 하는가?
- 중복 코드 발견
- 함수가 너무 길 때
- 클래스가 너무 클 때
- 매개변수가 너무 많을 때
- 기능 추가 전

## 주요 기법
- Extract Method (메서드 추출)
- Rename (이름 변경)
- Move Method (메서드 이동)
- Replace Conditional with Polymorphism
- Introduce Parameter Object

## 원칙
- 테스트 코드 먼저
- 작은 단계로 진행
- 자주 커밋
