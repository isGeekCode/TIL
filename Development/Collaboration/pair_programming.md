# Pair Programming (짝 프로그래밍)

> 두 명이 함께 코딩하기

**(예정중)**

## 개요
Pair Programming은 두 명의 개발자가 한 컴퓨터에서 함께 코드를 작성하는 협업 방식입니다.

## 역할
- **Driver** (운전자): 키보드를 잡고 코드 작성
- **Navigator** (네비게이터): 전체 방향 제시, 리뷰

## 방식
- **전통적**: 같은 공간, 한 컴퓨터
- **원격**: 화면 공유, Live Share

## 장점
- 즉각적인 코드 리뷰
- 지식 공유
- 버그 조기 발견
- 집중력 향상

## 단점
- 인력 2배 투입
- 피로도 증가
- 성격 차이

## 적용 시기
- 복잡한 기능 개발
- 신입 개발자 온보딩
- 중요한 버그 수정

## 변형
- Ping-Pong Programming (TDD와 결합)
- Mob Programming (3명 이상)
