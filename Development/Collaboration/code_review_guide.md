# Code Review Best Practices

> 효과적인 코드 리뷰 가이드

**(예정중)**

## 개요
코드 리뷰는 코드 품질을 높이고 지식을 공유하는 중요한 협업 활동입니다.

## 리뷰어 가이드

### 체크리스트
- 기능이 요구사항을 만족하는가?
- 버그나 논리적 오류는 없는가?
- 테스트 코드가 충분한가?
- 코드가 읽기 쉬운가?
- 보안 이슈는 없는가?

### 좋은 리뷰 방법
- 긍정적인 톤 유지
- 구체적인 피드백
- 질문 형태로 제안
- 칭찬도 함께

### 나쁜 리뷰
- "이거 왜 이렇게 했어요?"
- "이건 아닌 것 같은데"
- 세세한 것까지 지적

## 작성자 가이드
- PR 설명 명확히
- 스스로 먼저 리뷰
- 방어적이지 말기
- 피드백에 감사

## 리뷰 시간
- 24시간 이내 1차 리뷰
- 작은 PR: 30분 이내
- 큰 PR: 최대 1시간

## 도구
- GitHub PR
- GitLab MR
- Gerrit
