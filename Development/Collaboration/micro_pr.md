# Micro PR (Small Pull Request)

> 작고 자주 하는 PR 전략

**(예정중)**

## 개요
Micro PR은 작은 단위로 자주 Pull Request를 만드는 협업 전략입니다.

## 원칙
- 한 PR은 하나의 목적
- 변경 사항 200~400 라인 이하 권장
- 리뷰 시간 30분 이내
- 자주, 빠르게 머지

## 장점
- 빠른 코드 리뷰
- 버그 발견 용이
- 머지 충돌 감소
- 리뷰어 부담 감소
- 빠른 피드백

## 단점
- PR 개수 증가
- 문맥 파악 어려움 (해결: 설명 강화)

## 실천 방법
1. 기능을 작은 단위로 분리
2. 리팩토링과 기능 추가 분리
3. 명확한 PR 제목과 설명
4. Draft PR 활용

## 통계
- 10줄 미만: 리뷰 시간 ~10분
- 100줄: 리뷰 시간 ~30분
- 500줄 이상: 리뷰 품질 급격히 저하

## 관련 개념
- Trunk-Based Development
- Continuous Integration
