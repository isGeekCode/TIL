# DDD - 도메인 주도 설계

> Domain-Driven Design

**(예정중)**

## 개요
DDD는 복잡한 비즈니스 도메인을 소프트웨어로 모델링하는 설계 방법론입니다.

## 핵심 개념
- Entity (엔티티)
- Value Object (값 객체)
- Aggregate (집합체)
- Repository (리포지토리)
- Domain Service (도메인 서비스)

## 전략적 설계
- Bounded Context (경계 컨텍스트)
- Context Map (컨텍스트 맵)
- Ubiquitous Language (보편 언어)

## 주요 내용
- 도메인 모델링
- 계층 아키텍처
- 애그리게이트 설계
