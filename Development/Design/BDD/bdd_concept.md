# BDD - 행위 주도 개발

> Behavior-Driven Development

**(예정중)**

## 개요
BDD는 사용자의 행위(Behavior)를 중심으로 테스트를 작성하는 개발 방법론입니다.

## Given-When-Then 패턴
- **Given**: 주어진 상황
- **When**: 어떤 행위를 할 때
- **Then**: 예상되는 결과

## TDD와의 차이
- TDD: 개발자 관점, 코드 중심
- BDD: 사용자 관점, 행위 중심

## 주요 내용
- Gherkin 문법
- Cucumber, Jasmine 등 도구
- Feature 파일 작성
