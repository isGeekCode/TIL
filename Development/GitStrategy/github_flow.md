# GitHub Flow

> 간단하고 빠른 브랜치 전략

**(예정중)**

## 개요
GitHub Flow는 Git Flow를 단순화한 모델로, 지속적 배포에 적합합니다.

## 브랜치 종류
- **main** (또는 master): 항상 배포 가능
- **feature**: 기능/수정 브랜치

## 흐름
1. main에서 feature 브랜치 생성
2. 기능 개발 및 커밋
3. Pull Request 생성
4. 코드 리뷰 및 토론
5. 테스트 통과 후 main에 머지
6. 즉시 배포

## 특징
- 매우 간단함
- 지속적 배포(CD)에 최적
- main 브랜치는 항상 배포 가능 상태 유지

## 적합한 프로젝트
- 웹 서비스
- 빠른 반복 개발
- CI/CD 환경
