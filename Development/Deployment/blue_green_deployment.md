# Blue-Green Deployment

> 무중단 배포 전략

**(예정중)**

## 개요
Blue-Green Deployment는 두 개의 동일한 프로덕션 환경을 유지하며 무중단 배포를 실현하는 전략입니다.

## 작동 방식
1. **Blue**: 현재 운영 중인 환경
2. **Green**: 새 버전을 배포할 환경
3. Green 환경에 배포 및 테스트
4. 트래픽을 Blue → Green으로 전환
5. 문제 발생 시 즉시 Blue로 롤백

## 장점
- 무중단 배포
- 빠른 롤백 가능
- 배포 전 충분한 테스트

## 단점
- 인프라 비용 2배
- 데이터베이스 마이그레이션 복잡

## 적합한 환경
- 클라우드 환경
- 무중단이 중요한 서비스
