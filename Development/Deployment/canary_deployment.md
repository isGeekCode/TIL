# Canary Deployment

> 점진적 배포 전략

**(예정중)**

## 개요
Canary Deployment는 새 버전을 일부 사용자에게만 먼저 배포하고, 문제가 없으면 점진적으로 확대하는 전략입니다.

## 작동 방식
1. 전체 사용자의 5% → 새 버전
2. 모니터링 및 확인
3. 문제 없으면 10% → 20% → 50% → 100%
4. 문제 발생 시 즉시 롤백

## 장점
- 위험 최소화
- 실제 사용자 피드백
- 점진적 검증

## 단점
- 복잡한 모니터링 필요
- 트래픽 분산 인프라 필요

## 모니터링 지표
- 에러율
- 응답 시간
- 사용자 피드백

## 관련 개념
- Feature Flag
- A/B Testing
